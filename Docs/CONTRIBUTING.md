## 2. Contribution Workflow

### 1. Discuss Large Changes First
Before starting work on any major features, architectural refactors, or significant changes, please open an Issue to discuss it with the maintainers. This ensures your work aligns with the project roadmap and avoids duplicate effort.

### 2. Branching and Pull Requests
*   Fork the repository and create your branch from `main` (e.g., `feature/my-new-feature` or `bugfix/issue-123`).
*   Write clean, documented code and include unit tests where appropriate.
*   Ensure all tests pass locally before submitting.
*   Keep pull requests focused on a single change.

### 3. Code Style & Standards
*   Follow the style guidelines of the language you are writing in (e.g., standard Rust idioms and Swift API Design Guidelines).
*   Run linters and formatters (like `cargo fmt`) before submitting.
*   Write clear, descriptive commit messages.
